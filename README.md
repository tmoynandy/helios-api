## Authentication

Using **Clerk**. Configure these env vars in `.env` (and in your Kubernetes Secret at deploy time):

- `AUTH_ISSUER` — e.g. `https://your-instance.clerk.accounts.dev`
- `AUTH_JWKS_URL` — e.g. `https://your-instance.clerk.accounts.dev/.well-known/jwks.json`
- `AUTH_AUDIENCE` — optional; when set, the middleware rejects tokens with a mismatched `aud` claim.

Clerk issues RS256 JWTs. Find your instance's issuer and JWKS URL at dashboard.clerk.com → API Keys. `AUTH_AUDIENCE` is your Clerk application id (not required by Clerk but still validated when set).

Generated in `src/middleware/auth.ts`. Apply via `app.use(authRequired)` or per-route as a middleware argument.
